# Device UI

本文档定义设备端的页面结构和单键交互规则。

## 1. 总原则

设备主交互只使用一套简单的单键规则：

- `短按`：切换 / 移动
- `长按`：进入 / 确认 / 返回

V1 不使用双长按。

`B` 键不参与主交互流，只保留少量系统用途，例如进入配对。

## 2. 主界面

主界面有三种显示模式，短按循环切换：

### 2.1 图像模式

只显示宠物动画，适合远距离一眼查看。

### 2.2 混合模式

显示：

- 宠物动画
- 一行焦点摘要
- 一行状态摘要

这是推荐的默认模式。

### 2.3 信息模式

只显示文字和状态信息，例如：

- 当前主状态
- 焦点摘要
- session 状态摘要
- token 或网络状态

## 3. 主界面交互

在主界面中：

- `短按`：切换到下一个显示模式
- `长按`：进入菜单

三种模式循环切换：

- 图像模式 -> 混合模式 -> 信息模式 -> 图像模式

## 4. 菜单

菜单采用单列结构，短按移动，长按进入。

基础菜单建议包括：

- Brightness
- Volume
- Wi‑Fi
- Pair
- Info
- Help
- Back

## 5. 菜单交互

### 5.1 菜单首页

- `短按`：移动到下一个菜单项
- `长按`：进入当前菜单项

### 5.2 菜单子页

- `短按`：切换当前值或翻页
- `长按`：确认当前值并返回上一级

### 5.3 返回项

- 选中 `返回` 后长按，回到上一级
- 如果已经在菜单首页，选中 `返回` 后长按，回到主界面

## 6. 子菜单建议

### 6.1 Brightness

选项：

- Low
- Mid
- High
- Back

### 6.2 Volume

选项：

- Mute
- Low
- Mid
- High
- Back

### 6.3 Wi‑Fi

选项：

- Setup
- Clear
- Back

### 6.4 Pair

选项：

- Start
- Retry
- Back

进入配对后可显示：

- 当前 6 位绑定码
- 当前状态：等待绑定 / 已绑定 / 超时

### 6.5 Info

显示：

- `device_id`
- Wi‑Fi 状态
- MQTT 状态
- 固件版本
- 资源包版本

短按翻页，长按返回。

### 6.6 Help

显示：

- 如何配网
- 如何绑定
- 当前按键规则

短按翻页，长按返回。

## 7. 页面状态机

设备 UI 的最小状态机如下：

```text
主界面
  -> 短按切换三种显示模式
  -> 长按进入菜单首页

菜单首页
  -> 短按切换菜单项
  -> 长按进入子页

菜单子页
  -> 短按切换值或翻页
  -> 长按确认并返回
```

## 8. 选中态

菜单中必须有明确的当前选中项。

建议：

- 当前项前显示 `>`
- 当前值高亮
- `Back` 永远放在最后一项

## 9. 当前边界

本文件只定义页面结构和交互，不定义宠物动画和行为策略。
