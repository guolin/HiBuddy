# Buddy 宠物动画规范与设计绘画表

## 1. 目标

Buddy V1 的宠物动画分成两层：

- 资源层：画师提供固定帧
- 程序层：设备负责微位移、随机停顿、状态内变体和短时特效

这意味着宠物不会只是“固定帧死循环”，而是会在固定资源之上，通过程序产生更灵动的呼吸感、漂浮感和偶发小动作。

V1 仍然只有一个宠物角色，默认演示资源是单一小猴子。

## 2. 七个动画状态

| 编号  | 状态        | 用途                                   |
| ----- | ----------- | -------------------------------------- |
| `S00` | `sleep`     | 睡眠、面朝下 nap、低能量               |
| `S01` | `idle`      | 正常待机、轻微呼吸感                   |
| `S02` | `busy`      | 正在工作、专注执行                     |
| `S03` | `attention` | 需要用户注意                           |
| `S04` | `celebrate` | 完成任务后的短时开心状态               |
| `S05` | `dizzy`     | 错误、离线退化、粗暴摇晃               |
| `S06` | `heart`     | 友好互动、欢迎、恢复在线的短时正向特效 |
| `S07` | `bones`     |                                        |

## 3. 三属性对演出的影响

### mood

- 高：idle 更轻快，heart 更容易出现，celebrate 更完整
- 低：idle 更慢，退化时更容易呈现 dull / dizzy 感

### energy

- 高：idle 和 busy 的微位移更明显
- 低：整体节奏放慢，更容易进入 sleep

### focus

- 高：busy 更稳、更紧凑，随机偏移更小
- 低：busy 更容易回落到 idle，attention 更松散

## 4. Pet 页位移安全范围

当前 Pet 页绘制方式：

- 屏幕尺寸：`135x240`
- 宠物显示尺寸：`128x128`
- 当前绘制起点：`x=4, y=8`

因此安全位移范围必须保守：

- 横向推荐：`-2px ~ +2px`
- 纵向推荐：`0px ~ +4px`
- celebrate 可短时放宽到 `y +3px`
- 不允许大幅左右位移，避免裁切

结论：

- 横向只做轻微摆动
- 纵向主要负责呼吸感和漂浮感
- 不为位移单独重画资源，全部由程序完成

### 5.1 V1 最小资源需求

V1 最小必须画出来的状态组：

- `sleep base`
- `idle base`
- `idle blink`
- `idle tilt`
- `busy base`
- `busy focus`
- `attention base`
- `celebrate base`
- `dizzy base`
- `heart base`

如果时间有限，优先级顺序为：

1. `idle base`
2. `idle blink`
3. `busy base`
4. `sleep base`
5. `busy focus`
6. 其余状态

## 6. 程序行为表

这张表给开发实现程序层使用。

| 状态        | 帧间隔范围  | 停顿规则                 | 位移范围           | 变体触发条件              | 三属性影响                      | 是否短时态 |
| ----------- | ----------- | ------------------------ | ------------------ | ------------------------- | ------------------------------- | ---------- |
| `sleep`     | `350-500ms` | 每轮可有轻微长停顿       | `x 0`, `y 0~2`     | 极低概率 blink/rest tweak | energy 低时更慢                 | 否         |
| `idle`      | `220-320ms` | 有随机 pause，不等长循环 | `x -2~2`, `y 0~4`  | 定时尝试触发 blink / tilt | mood 高更轻快，energy 低更慢    | 否         |
| `busy`      | `120-180ms` | 轻微短停顿，不要拖沓     | `x -1~1`, `y 0~2`  | 定时触发 focus 变体       | focus 高更紧凑，focus 低更松    | 否         |
| `attention` | `140-220ms` | 短停顿后再回主节奏       | `x -1~1`, `y 0~2`  | 可偶发一次轻 tilt         | focus 低时更松，energy 低时更慢 | 否         |
| `celebrate` | `100-160ms` | 末帧可短停               | `x -2~2`, `y 0~3`  | 不切变体                  | mood 高时更完整                 | 是         |
| `dizzy`     | `160-240ms` | 不规则短停顿             | `x -2~2`, `y 0~2`  | 不切变体                  | mood 低时更容易进入             | 是         |
| `heart`     | `120-180ms` | 固定短时播放             | `x -1~1`, `y -1~2` | 不切变体                  | mood 高时更容易出现             | 是         |

## 7. 随机性规则

随机必须是“受控随机”，不能变成不可预测的噪声。

规则：

- 偶发小动作必须有最小间隔
- 同一 rare 动作不能连续重复
- `heart / celebrate / dizzy` 期间不再插入 idle/busy 变体
- 变体默认只持续 `1-2` 个循环

### 7.1 V1 实现建议

- `idle`
  - 随机停顿
  - 低概率 blink
  - 低概率 tilt
- `busy`
  - 低概率 focus 变体
- `sleep`
  - 极低概率 blink/rest tweak
- 其它状态
  - 先做程序位移和停顿，不强求复杂变体

## 8. 刷新策略

参考 `claude-desktop-buddy` 的低频局部刷新思路：

- 宠物动画按约 `150-220ms` 更新
- 属性计算按约 `1000ms` 更新
- 页面切换、菜单、guide 切换允许整页重绘
- 平时不要整屏每帧重画

区域职责：

- 宠物区：负责 sprite
- 文字区：负责状态文案
- session 点区：负责工作摘要
- 属性条区：只在 `Overview` 页显示

## 9. 状态映射规则

优先级如下：

1. `heartUntilMs` 有效时，优先 `heart`
2. `ScreenSleep`、面朝下、或 `energy` 很低时，优先 `sleep`
3. 摇晃晕眩或严重退化时，优先 `dizzy`
4. `requires_user=true` 或 `waiting`，优先 `attention`
5. `done` 进入短时 `celebrate`
6. `thinking / busy` 且 `focus` 足够高时，进入 `busy`
7. 其他情况进入 `idle`

补充规则：

- `heart` 播放后自动回落
- `celebrate` 短轮次后自动退出
- `sleep` 可由设备本地进入，不依赖上游强制下发

## 10. 页面语义

### Pet

- 主要显示宠物
- 显示 session 点
- 不显示三属性数值或 bar

### Overview

- 显示 `mood / energy / focus`
- 用 bar 表达，不显示数字
- 是宠物摘要页，不是设备页

### Info

- 只显示设备信息
- 包括电池、充电、Wi‑Fi、SSID、固件等
- 不显示三属性

## 11. V1 不做的内容

V1 不做：

- `affection` 正式运行态
- 云端同步宠物属性
- 多宠物切换
- 大量独立表情库
- 宠物属性直接通过 MQTT 下发
- 为每个程序位移单独重画资源

如果未来要加入养成感，可在 V2 新增慢变量 `bond / affection`，但不应和当前 `snapshot` 协议混在一起。
