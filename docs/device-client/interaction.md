# Buddy Device Client Interaction

本文档定义 `M5StickS3` 上的交互逻辑，包括：

- `A/B` 键使用原则
- 三个主界面切换规则
- 菜单结构
- 长按和短按定义
- 页面级状态机

## 1. 交互目标

设备交互要满足四个目标：

- 一眼可懂
- 单手可操作
- 低学习成本
- 不误触关键动作

由于设备屏幕小、按键少，V1 交互必须极简。

## 2. 按键设计原则

本方案使用两个实体键：

- `A`
- `B`

每个键支持：

- 短按
- 长按

### 2.1 时间定义

建议统一为：

- 短按：按下并在 `50ms ~ 500ms` 内释放
- 长按：持续按住 `>= 700ms`

`500ms ~ 700ms` 之间视为保留区间，避免误判。

## 3. 全局按键约定

### 3.1 `A` 键

`A` 键用于主导航。

- `A` 短按：下一个 / 切换 / 移动
- `A` 长按：进入 / 确认

### 3.2 `B` 键

`B` 键用于次级动作和返回。

- `B` 短按：返回 / 上一个 / 关闭提醒
- `B` 长按：快捷系统动作

### 3.3 全局规则

- 屏幕熄灭后第一次按键只负责唤醒
- 长按必须有进度反馈
- 关键动作不放在短按上
- 删除和清空操作必须二次确认

## 4. 三个主界面

主界面固定为三种：

- `Image`
- `Mixed`
- `Info`

推荐默认进入 `Mixed`。

默认主界面尽量少文字。

除菜单、帮助、设置类页面外，优先使用：

- 宠物表情
- 状态灯
- 小图标

而不是句子型文案。

### 4.1 Image

适合远距离查看。

显示：

- 大尺寸宠物
- 极少文字
- 当前主状态角标

### 4.2 Mixed

适合作为默认界面。

显示：

- 宠物
- session 状态灯
- 极简状态图标
- 小型网络状态角标

### 4.3 Info

适合排查和确认当前运行状态。

显示：

- 当前主状态
- session 灯阵列或计数
- Wi-Fi
- MQTT
- 设备信息
- token 概览

## 5. 主界面交互

在主界面中：

- `A` 短按：切换下一个主界面
- `A` 长按：打开菜单
- `B` 短按：切换上一个主界面
- `B` 长按：进入快捷系统页

三种主界面循环关系：

`Image -> Mixed -> Info -> Image`

`B` 短按按逆序切换：

`Image <- Mixed <- Info <- Image`

## 6. 快捷系统页

`B` 长按在主界面触发快捷系统页。

建议只包含高频动作：

- Pair
- Wi-Fi
- Brightness
- Restart
- Back

这样用户不需要每次都进入完整菜单层级。

## 6.1 IMU 快捷动作

V1 建议支持以下 IMU 交互：

- `shake`：触发临时 `dizzy` 演出
- `face-down`：进入睡眠或静音
- `pick-up`：唤醒屏幕

这些动作只用于轻交互和演出，不用于关键业务确认。

## 7. 菜单结构

完整菜单建议为单列结构：

- Brightness
- Volume
- Wi-Fi
- Pair
- Pet
- Info
- Help
- Restart
- Back

## 8. 菜单导航规则

### 8.1 菜单首页

- `A` 短按：移动到下一个菜单项
- `B` 短按：移动到上一个菜单项
- `A` 长按：进入当前菜单项
- `B` 长按：退出菜单返回主界面

### 8.2 菜单子页

- `A` 短按：切换当前值或翻到下一项
- `B` 短按：切换到上一项或上一页
- `A` 长按：确认当前值
- `B` 长按：返回上一级

## 9. 为什么这样分配

这种分配比“一个键全部做完”更自然：

- `A` 负责前进
- `B` 负责返回
- 两个键短按都可用于遍历
- 两个键长按职责不同，不会冲突

这样用户很快能形成肌肉记忆。

## 10. 重点页面交互

### 10.1 Brightness

选项：

- `Low`
- `Mid`
- `High`
- `Auto`
- `Back`

规则：

- `A/B` 短按切换
- `A` 长按保存
- `B` 长按返回

### 10.2 Volume

选项：

- `Mute`
- `Low`
- `Mid`
- `High`
- `Back`

### 10.3 Wi-Fi

选项：

- `Setup`
- `Reconnect`
- `Clear Saved`
- `Status`
- `Back`

规则：

- `Setup` 进入配网页
- `Reconnect` 立即重连
- `Clear Saved` 需要确认

### 10.4 Pair

选项：

- `Start Pairing`
- `Refresh Code`
- `Unbind and Re-pair`
- `Status`
- `Back`

规则：

- 已绑定状态下默认不重复开放绑定
- `Unbind and Re-pair` 需要确认页

### 10.5 Pet

V1 如果只有一个宠物，此页也保留。

显示：

- 当前宠物名
- 当前版本
- 当前资源状态
- 重新下载
- Back

### 10.6 Info

分页显示：

- `device_id`
- firmware version
- asset version
- Wi-Fi RSSI
- MQTT state
- last snapshot time

### 10.7 Help

分页显示：

- 如何配网
- 如何绑定
- 按键说明
- 问题恢复方式

## 11. 临时提醒交互

临时提醒包括：

- `waiting`
- `error`
- `done`
- `stale`

提醒页规则：

- 自动覆盖当前主界面
- `B` 短按可立即关闭
- 若用户不操作，数秒后自动返回
- 返回时回到提醒前所在页面

### 11.1 waiting 的 V1 定义

`waiting` 在 V1 中表示：

- 当前有需要用户处理的事情
- 宠物进入提醒态
- 设备引导用户回到桌面或 Web 继续处理

V1 不要求设备显示完整审批内容。

V1 也不要求设备直接发送审批结果。

### 11.2 设备审批作为 V2 预留

如果后续加入设备直接审批，建议采用：

- `A` 短按：Approve
- `B` 短按：Deny

不建议使用：

- 同一个按键短按通过、长按拒绝

原因：

- 易误触
- 破坏键位直觉
- 不利于高风险动作区分

但这一能力属于 V2，不属于 V1 默认范围。

## 12. 熄屏与唤醒

建议增加低打扰规则：

- 长时间无操作可熄屏
- 新 `snapshot` 到达可唤醒屏幕
- 关键提醒可临时点亮屏幕

熄屏后：

- 任意键第一次按下只唤醒
- 不执行页面切换

## 13. 页面状态机

```text
主界面
  -> A短按: 下一个主界面
  -> B短按: 上一个主界面
  -> A长按: 完整菜单
  -> B长按: 快捷系统页

完整菜单
  -> A/B短按: 移动
  -> A长按: 进入
  -> B长按: 返回主界面

子页面
  -> A/B短按: 切换值/翻页
  -> A长按: 确认
  -> B长按: 返回

临时提醒
  -> 自动超时返回
  -> B短按: 立即关闭
```

## 14. V1 交互边界

V1 不建议加入：

- 双击
- 双长按
- 复杂按键组合
- 长文本编辑
- 设备侧输入复杂文本
- V1 设备侧直接审批
